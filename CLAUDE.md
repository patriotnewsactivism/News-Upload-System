## Architecture Overview

This is a **pnpm workspace monorepo** with three main areas:

- **`artifacts/`**: Deployable applications (Express API server, React frontends, etc.)
- **`lib/`**: Shared libraries and generated code (database layer, API schemas, API client)
- **`scripts/`**: Utility scripts that can import workspace packages

### Core Tech Stack

- **Monorepo**: pnpm workspaces
- **Language**: TypeScript 5.9 with composite projects
- **Backend**: Express 5 API server with Drizzle ORM (PostgreSQL)
- **Validation**: Zod (generated from OpenAPI spec)
- **API Codegen**: Orval (from `lib/api-spec/openapi.yaml`)
- **Database**: PostgreSQL + Drizzle ORM with `drizzle-zod` for insert schemas
- **Build**: esbuild for production bundles (CJS), tsx for dev/scripts
- **UI**: React 19 + Tailwind CSS (in frontend artifacts)

## Key Development Commands

### Workspace-level Commands

```bash
pnpm run build              # typecheck + build all packages
pnpm run typecheck          # tsc --build --emitDeclarationOnly (always run from root)
pnpm install                # install dependencies
```

### API Server (`@workspace/api-server`)

```bash
pnpm --filter @workspace/api-server run dev       # start dev server (auto-reload via tsx)
pnpm --filter @workspace/api-server run build     # production bundle (dist/index.cjs)
pnpm --filter @workspace/api-server run typecheck # typecheck only
```

### Database (`@workspace/db`)

```bash
pnpm --filter @workspace/db run push              # apply pending migrations
pnpm --filter @workspace/db run push-force        # force apply migrations (use if push fails)
```

### API Codegen (`@workspace/api-spec`)

```bash
pnpm --filter @workspace/api-spec run codegen     # regenerate schemas and React Query hooks
```

### Utility Scripts (`@workspace/scripts`)

```bash
pnpm --filter @workspace/scripts run <script-name>  # run a script from scripts/src/
```

## TypeScript & Composite Projects

**Key principle**: Always typecheck from the root, not individual packages.

- Each package has `composite: true` in its `tsconfig.json` (inherited from `tsconfig.base.json`)
- Root `tsconfig.json` lists all packages as project references
- When package A depends on B, A's `tsconfig.json` must list B in `references` array
- **Always run**: `pnpm run typecheck` (which runs `tsc --build --emitDeclarationOnly`)
- **Never run** `tsc` inside a single package unless all dependencies are built first
- `emitDeclarationOnly` mode: we emit only `.d.ts` files during typecheck; actual JS bundling is handled by esbuild/tsx

## Package Architecture

### `artifacts/api-server` (`@workspace/api-server`)

Express 5 API with routes that handle news articles, newsletter subscriptions, and content uploading.

- **Entry**: `src/index.ts` — reads `PORT` env var, starts server
- **App setup**: `src/app.ts` — mounts CORS, JSON/urlencoded parsing, routes at `/api`
- **Routes**: `src/routes/index.ts` mounts sub-routers; individual routes in `src/routes/*.ts`
  - `health.ts` — `GET /api/health`
  - `articles.ts` — `GET /api/articles`, `POST /api/articles`, etc.
- **Validation**: Uses `@workspace/api-zod` for request/response validation
- **Database**: Uses `@workspace/db` for persistence
- **Build**: esbuild bundles allowlist of deps (express, cors, pg, drizzle-orm, zod, etc.) and externalizes the rest

### `lib/db` (`@workspace/db`)

Drizzle ORM database layer.

- **Exports**: Pool and Drizzle client instance, schema models
- **Entry**: `src/index.ts` — creates and exports `pool` (PG connection pool) and `db` (Drizzle instance)
- **Schema**: `src/schema/` — table definitions (e.g., `articles.ts` with `drizzle-zod` insert schemas)
- **Config**: `drizzle.config.ts` — Drizzle Kit config (requires `DATABASE_URL` from environment)
- **Migrations**: In production, Replit handles migrations on publish. In dev, use `pnpm --filter @workspace/db run push`

### `lib/api-spec` (`@workspace/api-spec`)

OpenAPI 3.1 spec and Orval codegen configuration.

- **Spec**: `openapi.yaml` — single source of truth for API contracts
- **Config**: `orval.config.ts` — Orval setup that generates code into sibling packages
- **Generated output**:
  - `lib/api-client-react/src/generated/` — React Query hooks + fetch client
  - `lib/api-zod/src/generated/` — Zod schemas
- **Workflow**: Update `openapi.yaml` → run `pnpm --filter @workspace/api-spec run codegen` → generated code auto-updates

### `lib/api-zod` (`@workspace/api-zod`)

Generated Zod schemas (e.g., `HealthCheckResponse`, `CreateArticleRequest`).

- Used by `api-server` for request/response validation
- Regenerated by Orval from `openapi.yaml`
- Do not manually edit; edit the spec and regenerate

### `lib/api-client-react` (`@workspace/api-client-react`)

Generated React Query hooks (e.g., `useHealthCheck`, `healthCheck()`) and fetch client.

- Regenerated by Orval from `openapi.yaml`
- Custom fetch: `src/custom-fetch.ts` can be extended for auth/interceptors

### `artifacts/we-the-people-news` (Frontend)

Vite + React frontend for the news site.

- Uses `@workspace/api-client-react` hooks to call the API
- Tailwind CSS + shadcn/ui components

### `artifacts/mockup-sandbox` (Mockup Preview)

Vite + React sandbox for previewing UI mockups and design components.

## Workflow for Common Tasks

### Adding a New API Endpoint

1. Update `lib/api-spec/openapi.yaml` with the new endpoint definition
2. Run `pnpm --filter @workspace/api-spec run codegen`
3. Implement the route in `artifacts/api-server/src/routes/<feature>.ts`
   - Use generated schemas from `@workspace/api-zod` for validation
   - Use `@workspace/db` for queries
4. Optionally use generated hooks in frontends from `@workspace/api-client-react`

### Modifying the Database Schema

1. Edit table definitions in `lib/db/src/schema/<table>.ts`
2. Run `pnpm run typecheck` to ensure types are correct
3. Run `pnpm --filter @workspace/db run push` to apply migrations to the database

### Deploying Changes

1. `pnpm run build` — full typecheck + bundle
2. Commit and push to main
3. Replit auto-deploys from the main branch

## Don Matthews Investigative Reporting Workflow

### Mission Focus

We The People News focuses on exposing and reporting on:
- **Corrupt Law Enforcement** — police abuse, misconduct, cover-ups, federal agent overreach
- **Public Officials** — corruption, embezzlement, bribery, fraud by government officials
- **Activism** — grassroots movements, civil rights actions, community organizing
- **Court Cases** — ongoing litigation and closed case analysis

### Quick Publish Workflow

1. **Upload & Parse** (`/admin` → "Upload Content" tab)
   - Paste raw HTML, plain text, evidence notes, research material, or bullet points
   - Select content type:
     - **Plain Text**: Standard article content
     - **HTML**: Formatted content with markup
     - **Evidence/Research Notes**: Case documents, evidence, analysis with optional context
   - System automatically:
     - Parses and cleans content
     - Generates article title (from first sentence or opening)
     - Creates summary (truncated to 200 chars)
     - Infers category using keyword matching:
       - *Corrupt Law Enforcement*: police, law enforcement, FBI, DOJ, abuse, brutality
       - *Public Officials*: politician, mayor, governor, embezzlement, bribery, fraud
       - *Court Cases - Open*: pending, active, ongoing litigation terms
       - *Court Cases - Closed*: past cases, closed litigation
       - *Activism*: protest, movement, civil rights, advocacy
     - Extracts 6-8 relevant tags (Police Abuse, Corruption, Civil Rights, Justice, etc.)
     - Calculates read time
   - Review generated draft preview
   - Click "Open in Editor" to refine

2. **Write / Edit** (`/admin` → "Write / Edit" tab)
   - Full article editor with:
     - **Title**: Article headline
     - **Summary**: 2–3 sentence synopsis
     - **Content**: HTML-supported body (use `<p>`, `<strong>`, `<em>`, `<blockquote>` for structure)
     - **Category**: 5 WTP-specific categories (see above)
     - **Tags**: Comma-separated keywords for discoverability
     - **Image URL**: Link to cover image
     - **Featured**: Toggle for homepage prominence
     - **Status**: Draft or Published
     - **Publish At**: (Optional) Schedule publication for specific date/time
   - Real-time validation and feedback
   - Auto-calculates read time

3. **Article Management** (`/admin` → "Articles" tab)
   - Table view of all articles with:
     - Title and excerpt
     - Category badge
     - Publication status (Published/Draft)
     - Featured indicator (⭐)
     - Creation date
   - Actions: View, Edit, Delete
   - Bulk operations available via API

### Advanced Features

#### Scheduling & Bulk Publishing
- **Schedule individual articles** for future publication (editor form)
- **Bulk publish** multiple articles at once via `/api/articles/bulk-publish`:
  ```bash
  POST /api/articles/bulk-publish
  { "articleIds": [1,2,3], "publishAt": "2024-03-20T09:00:00Z" }
  ```

#### Viral Metrics & Analytics
- **Automatic tracking** of:
  - Views per article
  - Share count
  - Viral Score (views + shares×10)
- **Update analytics** via `/api/articles/{id}/analytics`:
  ```bash
  PUT /api/articles/1/analytics
  { "views": 1, "shares": 0 }
  ```

### Content Upload Features

- **Multiple input formats**: HTML, plain text, evidence/research notes
- **Smart parsing**: Cleans HTML, preserves structure, handles entities
- **Evidence mode**: Captures raw evidence with optional analysis context
- **Automatic metadata generation**: Titles, summaries, categories, tags
- **SEO optimization**: Auto-generates meta titles and descriptions

### Frontend Display

- **Home page** (`/`): All published articles sorted newest-first
- **Category filtering**: View articles by category (`?category=Corruption`)
- **Featured articles**: Prominent homepage placement
- **Article pages** (`/article/:id`): Full text with read time estimate
- **Newsletter signup**: Build reader base for campaigns

### Key Fields & Defaults

- **Author**: Always "Don Matthews"
- **Default Category**: Public Officials (most common)
- **Read Time**: Auto-calculated (words ÷ 200/min)
- **Slug**: Auto-generated from title + timestamp for uniqueness
- **Scheduling**: Optional scheduled publication for optimal reach
- **Viral Metrics**: Automatically tracked (views, shares, score)

## Important Notes

- **Do not manually edit generated files** in `lib/api-zod/src/generated/` or `lib/api-client-react/src/generated/`. These are overwritten by Orval. After schema changes, run `pnpm --filter @workspace/api-spec run codegen`.
- **Always typecheck from root** before committing: `pnpm run typecheck` or `npx tsc --build`.
- **Database migrations**: Run `pnpm --filter @workspace/db run push` to apply pending migrations.
- **Build bundles**: API server production build (`pnpm --filter @workspace/api-server run build`) creates `dist/index.cjs`.
- **Admin password**: Configure via `VITE_ADMIN_PASSWORD` env var in production.
- **Content types**: Support HTML, plain text, and evidence/research note uploads with automatic parsing.
- **Category inference**: Uses keyword matching; update `inferCategory()` in `artifacts/api-server/src/routes/articles.ts` to customize.
- **Viral metrics**: Analytics tracked via `views`, `shares`, and computed `viralScore` (views + shares×10).
